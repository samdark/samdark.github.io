SMS Security and Alternatives
Mar 6, 2017
2 minutes read

The idea of two factor authentication is simple. One may get your password by fiddling with your wireless network connection but that should not give attacker an access to the system because login process asks for additional confirmation. This confirmation, a second factor, is a short token that is nowadays typically sent via SMS.

SMS has certain pros. There is no need to install any special applications or being connected to Internet. It works even on Nokia 3310. Everything’s great, right? Well, no.

What’s wrong with SMS?

The problem is that SMS is not a secure channel. 2016 clearly proved that. Details are summed very well in Fortune’s “Time Is Running Out For This Popular Online Security Technique” article.

So knowing your password and having access to your SMS (yes, that’s still not trivial) attacker can get your account.

Are there safer channels? Yes. Definitely.

One time passwords

One time passwords are tokens that could be used only once and that change often. These are not vulnerable to replay attack and are commonly used as a second authentication factor.

Tokens could be obtained either from hardware device such as YubiKey or software such as Google Authenticator.

There are two common kinds of one time passwords: HOTP (HMAC-based one time password) and TOTP (Time-based one time password). In both cases each one time password is an sha1 of a shared secret and a moving factor. Subsequent passwords are different each time.

  • In case of HOTP a moving factor is a counter. Each password generation increases the counter.
  • In case of TOTP a moving factor is unix time.

Since HOTP may be valid for a long time and TOTP is changed often by itself, TOTP is considered being safer.
