Digging into intl and ICU
Mar 7, 2017
1 minute read

Yii framework is used worldwide and mainly outside US. Therefore intenationalized applications are a common requirement and we needed a solid internationalization and localization components.

Such components are often limited to message translation but that is not enough. One needs things such as world countriy lists in all world languages, timezones, proper plural forms etc.

A library that can handle all that is intl but it’s used unfairly rare. The reasons are:

  • Bugs
  • Extra requirements
  • Insufficient docs

In Yii framework we’ve solved all three. Let’s review the points above one by one.

Bugs

intl PHP extension is based on ICU4c library which is used in many software projects including MacOS, …

Extra requirements

intl requires you to install extra stuff. First, a PECL PHP extension itself. Second, up to date ICU data.

Insufficient docs

Despite being used for many projects and despite being documented, ICU documentation is awful. There’s no complete description of syntax and no complete description of features. When documenting Yii features I’ve checked ICU sources multiple times before realizing all its features.
