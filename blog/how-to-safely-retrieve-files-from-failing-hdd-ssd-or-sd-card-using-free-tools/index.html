How to safely retrieve files from failing HDD, SSD or SD card using free tools
Mar 4, 2017
3 minutes read

All the drives we’re using to store our precious photos and videos aren’t 100% reliable. Everybody knows that and know that there’s a need to back everything up but even if you’re backing everything up, there are cases when drive fails and you have nowhere to get its contents from aside from the drive itself.

It happened to me recently. 23 GB SD card from a trip almost full of very precious shots was corrupted when inserted into cheap PC card reader.

As a result both Windows PC and Mac weren’t able to read anything from the card and were prompting to format it instead. After trying lots of commercial software designed specifically for recovering photos and files from SD cards I was lost. None worked well. One of the tools was able to recover some photos. About 2% of what was there initially.

After that I’ve found out how to do it properly and, surprisingly, for free.

First of all, drive may degrade so it’s not a good idea to try reading from it repeatedly. It may not hold it and die entirely. So we need a tool to copy entire drive to an image file on a local disk byte by byte reading as much as we can. Such a tool exists and is called GNU Ddrescue. The only con of it is that it requires MacOS or Linux to run but it’s not a problem since there are live CDs available. In particular, there’s Knoppix which has all the tools on board.

So we’re opening console and trying to find out which drives we have. In order to do it use fdisk -l on Linux and /Applications/Utilities/diskutil list on MacOS. On MacOS prefer disk names which start with /dev/r which are raw disks with buffer-less communication.

Now, as you know drive names:

# get most of the error-free data quickly
./ddrescue -n /dev/old_disk /mnt/sdb1/bad_drive.img rescued.log
# then attempt to get data which is in bad areas
./ddrescue -r 1 /dev/old_disk /mnt/sdb1/bad_drive.img rescued.log

/mnt/sdb1/bad_drive.img is a path to image file to create. Obviously, it should be on another disk and you should have at least same amount of space available as the side of original disk you’re restoring.

As it’s done, we need to actually get our data from the disk image we have. There’s another tool for that called PhotoRec. Again, it’s a console tool.

Launch it as ./photorec /mnt/sdb1/bad_drive.img then follow instructions. At this point you can try various options as many times as you want since image file isn’t affected and is on a well working drive so there’s little chance to lose data because of messing with original drive.

That’s it. Using these two tools I was able to recover all of the precious photos from corrupted SD card.
